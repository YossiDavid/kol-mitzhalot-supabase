# נתוני בדיקה לתבנית SendGrid — הצעת שידוך

קובץ: [`shidduch-offer.test-data.json`](./shidduch-offer.test-data.json)

## איך לבדוק בעורך SendGrid

1. פתחו **Dynamic Templates** → בחרו את תבנית `shidduch-offer` (או השם שנתתם).
2. בעורך התבנית, חפשו שדה **Test Data** / **Substitution tags** / הדמיית תצוגה (השם משתנה בין גרסאות).
3. הדביקו **רק אובייקט JSON אחד** — לא את כל הקובץ עם המפתחות `both` / `groom_only` / `bride_only`.

### דוגמה — להעתיק את זה לבדיקת «לשני הצדדים»

העתיקו את התוכן **בין הסוגריים** של `both` (כולל הסוגריים המסולסלים של האובייקט), כלומר מה-`{` הראשון אחרי `"both":` ועד ה-`}` המתאים:

```json
{
  "groom_name": "ישראל ישראלי",
  "bride_name": "לאה לוי",
  "shadchan_name": "משה כהן",
  "offer_url": "https://example.com/app/shidduchim/00000000-0000-4000-8000-000000000001",
  "note_for_groom": "שלום,\nנראה התאמה טובה בתחום הלימודים ובמשפחה. מומלץ לבדוק התאמת קהילה.",
  "note_for_bride": "המיועד מתאים בטווח הגיל ובכיוון החיים שצוין בפרופיל.",
  "is_both": true,
  "is_groom_only": false,
  "is_bride_only": false
}
```

### לבדיקת «רק מיועד» או «רק מיועדת»

השתמשו באובייקטים `groom_only` או `bride_only` מאותו קובץ — שוב, רק האובייקט הפנימי (רמת השדות שמופיעים בתבנית), בלי המפתח החיצוני.

## הערה

ב־SendGrid חייבים בוליאנים אמיתיים (`true` / `false`), לא מחרוזות.
